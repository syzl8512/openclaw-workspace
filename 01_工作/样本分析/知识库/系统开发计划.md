## 三、向量数据库设计

### 3.1 Ubuntu Supabase 存储结构

**数据库位置**：Ubuntu虚拟机

**表结构**：

```python
# samples 表
{
    "id": "uuid",
    "title": "湖北省政府工作报告2026",
    "category": "汇报材料",
    "direction": "向上",
    "industry": "政府",
    "source_level": "省级",
    "data_sensitivity": "public",  # "public" 或 "sensitive"
    "tags": ["新型工业化", "科技创新"],
    "content": "全文内容...",  # 分 chunk 存储
    "summary": "核心要点摘要",
    "metrics": "关键量化指标",
    "embedding": [0.123, -0.456, ...],  # 1024维向量
    "embedding_model": "jina",  # "jina" 或 "bge"
    "skill_path": "skills/writing/.../skill.md",
    "file_hash": "sha256:xxxxx",  # 用于缓存校验
    "created_at": "2026-03-01",
    "usage_count": 0,
    "rating": None
}
```

### 3.2 Tag标签索引（补充向量检索）

**索引文件**：`tags_index.json`

```json
{
    "categories": {
        "文档类型": ["汇报材料", "通知公文", "分析报告", "方案模板", "领导发言"],
        "沟通方向": ["向上", "平级", "向下"],
        "行业领域": ["工业", "农业", "科技", "金融", "医疗", "教育"],
        "来源层级": ["集团", "省公司", "市公司", "县政府", "外部"],
        "数据敏感度": ["public", "sensitive"]
    },
    "tag_index": {
        "新型工业化": ["sample_001", "sample_003", "sample_007"],
        "科技创新": ["sample_001", "sample_005"],
        "5G基站": ["sample_002", "sample_006"]
    }
}
```

### 3.3 检索策略

**双重检索机制**：
1. **Tag过滤** → 快速缩小范围
2. **向量相似度** → 语义匹配
3. **Rerank** → 综合排序

## 四、飞书文件处理流程

### 4.1 飞书文件接收机制

```
用户发送文件到飞书群 → 用户明确触发分析 → OpenClaw接收 → 自动下载到本地待处理
```

**触发条件**：
1. 用户输入：`@机器人 分析此文件`
2. 用户输入：`/样本分析 [文件]`
3. 用户发送文件 + 说"分析"

**不自动监听**：避免无关文件干扰分析

### 4.2 飞书输出机制

```
分析完成 → 生成Skill → 推送飞书群 → 消息卡片展示
```

- 推送通知到飞书群（带文件）
- 支持反馈按钮修正

## 六、指令体系

### 6.1 用户可用指令

```
/样本分析 [文件]              # 快速模式（4维）
/样本分析 [文件] --deep       # 深度模式（16维）
/样本搜索 [关键词]            # 检索样本库
/样本入库 [文件]              # 将当前文档加入样本库
/Skill加载 [类型]             # 加载对应Skill模板
/飞书同步                     # 手动触发飞书同步
```

### 6.2 自动触发

| 触发条件 | 自动动作 |
|----------|----------|
| 用户明确触发分析 | 下载 → 分析 → 归档 |
| 新增样本到库 | 向量化 → 索引 |
| 每周定时 | 效果评估 → 优化建议 |
