## 一、业务背景

### 1.1 问题陈述

当前公文编写面临以下挑战：

1. **素材分散** - 样本文件散落在各处，缺乏统一管理
2. **检索困难** - 无法快速找到相关参考素材
3. **重复劳动** - 相同类型的文档每次重新构思
4. **风格不一** - 不同人写的文档风格差异大
5. **数据提取难** - Excel/图表中的经营数据难以复用

### 1.2 业务目标

| 目标 | 量化指标 |
|------|----------|
| 样本集中管理 | 90%以上样本入库 |
| 快速检索 | 3秒内找到相关素材 |
| 复用率提升 | 60%以上文档基于样本生成 |
| 风格统一 | 标准化Skill覆盖率80% |
| 数据提取 | 90%以上表格/图表自动识别 |

## 二、需求调研

### 2.1 用户场景

**场景1：政策文件分析**
- 用户收到/看到一份政府文件
- 需要提取关键信息（背景/目标/任务/数据）
- 需要匹配联通产品机会
- 需要生成汇报素材

**场景2：经营数据分析**
- 用户收到一份Excel经营数据
- 需要识别表格结构和图表
- 需要提取关键指标和趋势
- 需要生成分析报告

**场景3：历史样本复用**
- 用户需要编写同类文档
- 需要快速检索历史样本
- 需要加载对应的写作Skill
- 需要提取相关数据和表述

### 2.2 核心需求

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| R-001 | 支持飞书文件接收（明确触发才分析） | P0 |
| R-002 | 支持PDF/Word/Excel源文件解析 | P0 |
| R-003 | 支持表格和图表识别（MinerU） | P0 |
| R-004 | 16维度标准化分析框架 | P0 |
| R-005 | 自动生成可复用Skill模板 | P0 |
| R-006 | 向量语义检索（混合架构） | P1 |
| R-007 | Tag标签分类索引 | P1 |
| R-008 | 自动归档到知识库 | P1 |
| R-009 | 使用效果追踪与迭代 | P2 |

### 2.3 关键约束

1. **明确触发原则**：飞书发送文件默认不分析，需用户明确指令才启动
2. **数据敏感**：经营数据涉及公司机密，需本地处理和存储
3. **离线优先**：主要功能本地运行，敏感数据不上云
4. **渐进式**：先搭建基础框架，再逐步智能化
5. **混合架构**：公开数据用云API，敏感数据用本地/私有部署

## 三、功能需求

### 3.1 文件接收模块

**输入源**：
- 飞书群文件（需明确触发）
- 本地目录放入
- 手动指定文件路径

**触发方式**：
```
用户：分析这个文件
用户：样本分析 /Users/xxx/文件.pdf
@机器人 分析此文件
```

**不支持自动监听**：避免无关文件干扰分析

### 3.2 文档解析模块

**支持格式**：

| 格式 | 解析方式 | 表格识别 | 图表识别 |
|------|----------|----------|----------|
| PDF | MinerU API | ✅ | ✅（作为图片） |
| Word | MinerU API | ✅ | ✅ |
| Excel | openpyxl/xlsx skill | ✅ 原生 | ⚠️ 基础识别 |
| Markdown | 直接读取 | ✅ | ❌ |

**降级方案**：如MinerU解析失败，自动降级到纯文本/OCR处理

### 3.3 数据分析模块（针对Excel/经营数据）

**分析维度**：

| 维度 | 说明 | 提取内容 |
|------|------|----------|
| 基础信息 | 文件/Sheet名称 | 标题、时间范围 |
| 表格结构 | 表头、数据区域 | 行列数、类型 |
| 关键指标 | 核心KPI | 收入、增长率、排名 |
| 趋势数据 | 时间序列 | 月度/季度变化 |
| 对比分析 | 横向/纵向对比 | 部门/区域/行业对比 |
| 图表识别 | 图表类型+数据 | 折线/柱状/饼图数据 |

### 3.4 梯度分析模式

| 模式 | 维度 | 耗时 | 适用场景 |
|------|------|------|----------|
| **快速模式** | 4维（背景/目标/任务/数据） | ~30秒 | 日常文件筛选 |
| **深度模式** | 16维（完整分析） | ~90秒 | 重点政策/高价值样本 |

**触发方式**：
- 默认快速模式
- 用户指定 `--deep` 时执行深度模式

### 3.5 16维分析模块

| 维度 | 分析内容 |
|------|----------|
| 1 | 政策背景/业务背景 |
| 2 | 核心目标 |
| 3 | 重点任务 |
| 4 | AI/数字化亮点 |
| 5 | 产业/业务影响 |
| 6 | 投资/预算机会 |
| 7 | 联通产品匹配 |
| 8 | 要点总结 |
| 9 | 量化指标 |
| 10 | 执行路径 |
| 11 | 风险评估 |
| 12 | 历史对比 |
| 13 | 区域/部门分析 |
| 14 | 时间节点 |
| 15 | 竞品/对标分析 |
| 16 | 可行性评估 |

### 3.6 Skill生成模块

**生成内容**：
- 文档结构模板
- 话术规范（开头/过渡/结尾）
- 必备要素清单
- 特色表达示例
- 量化指标参考

**输出格式**：符合OpenClaw metadata规范的Markdown格式

### 3.7 向量检索模块

**检索策略**：
1. **关键词检索** - 精确匹配
2. **Tag筛选** - 分类过滤
3. **向量语义检索** - 语义相似度
4. **组合检索** - 多条件组合

**缓存机制**：Hash校验避免重复解析（同名文件+内容Hash一致时跳过）

### 3.8 归档模块

**归档规则**：
- 源文件 → 01_原始样本/
- 分析结果 → 02_分析结果/
- 向量数据 → 03_向量库/
- Skill文档 → 05_Skill库/
- 清理临时文件

**Archive机制**：重要样本分析完成后，自动生成JSON镜像备份

│                              │                                   │
│                              ▼                                   │

          ▼                                         ▼

   └──────────────┘                          └──────────────┘
          │                                         │

### 4.2 数据流向

| 数据类型 | 向量化方式 | 存储位置 |
|----------|------------|----------|
| 公开政策文件 | jina API | Ubuntu Supabase |
| 经营数据（敏感） | bge-small-zh | Ubuntu Supabase |
| 临时文件 | - | Mac Mini本地 |

### 4.3 资源需求

| 组件 | 部署位置 | 内存 | 硬盘 | 状态 |
|------|----------|------|------|------|
| OpenClaw | Mac Mini | - | - | 已运行 |
| jina API | 云端 | - | - | 已配置 |
| Supabase | Ubuntu | 2GB | 100GB+ | 需部署 |
| bge-small-zh | Ubuntu | 500MB | 100MB | 需部署 |

## 五、非功能需求

### 5.1 性能需求

| 指标 | 快速模式 | 深度模式 |
|------|----------|----------|
| 文件解析 | < 15秒 | < 15秒 |
| 维度分析 | < 15秒 | < 75秒 |
| 向量检索 | < 3秒 | < 3秒 |
| Skill生成 | < 15秒 | < 15秒 |

### 5.2 安全需求

- 经营数据全程本地处理，不上传云端
- 敏感数据使用bge模型向量化
- Supabase仅存储向量和元数据
- Archive JSON镜像防止飞书消息过期

## 六、实施路径（优先级优化）

| 优先级 | 阶段 | 内容 | 依赖 |
|--------|------|------|------|
| **P0** | 基础架构 | 飞书接收→本地存储→简单文本分析 | 无需新部署 |
