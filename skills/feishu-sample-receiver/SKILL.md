# 飞书样本文件接收器

负责接收飞书群文件，明确触发后下载到本地待处理目录。

## 功能

1. **文件监听**：监听飞书群文件消息
2. **明确触发**：仅在用户明确要求分析时处理
3. **文件下载**：下载文件到本地待处理目录
4. **触发分析**：自动触发样本分析流程

## 触发条件

用户发送文件到飞书群，并明确触发：

```
用户：分析这个文件
用户：/样本分析 [文件名]
用户：样本分析 [文件名]
```

## 工作流程

```
飞书群收到文件
       │
       ▼
用户明确触发分析
       │
       ▼
验证文件格式（PDF/Word/Excel/Markdown）
       │
       ▼
下载到本地 00_临时/待处理/
       │
       ▼
记录文件元信息
       │
       ▼
触发样本分析流程
```

## 文件类型支持

| 类型 | 扩展名 | 说明 |
|------|--------|------|
| PDF | .pdf | 政策文件、报告 |
| Word | .docx | 文档材料 |
| Excel | .xlsx, .xls | 经营数据 |
| Markdown | .md | 笔记、稿件 |

## 存储路径

```
/Users/agent/Documents/syzl8512/01_工作/样本分析/
├── 00_临时/
│   ├── 待处理/          ← 下载到这里
│   ├── 处理中/
│   └── 完成/
└── 01_知识库/
    └── 01_原始样本/    ← 分析后归档到这里
```

## 元信息记录

下载文件时记录：

```json
{
  "file_id": "飞书文件ID",
  "file_name": "原始文件名",
  "file_type": "pdf/word/excel/markdown",
  "file_size": 1024000,
  "download_time": "2026-03-01T12:00:00Z",
  "source": "飞书群",
  "chat_id": "飞书群ID",
  "message_id": "消息ID",
  "status": "pending_analysis"
}
```

## 使用示例

### 场景1：飞书群分析文件

```markdown
用户：在群里发送文件 + "分析这个文件"

系统：✅ 收到文件：[文件名]
     开始下载...
     ✅ 下载完成
     开始分析...
```

### 场景2：手动指定文件

```markdown
用户：/样本分析 /Users/agent/Documents/xxx.pdf

系统：✅ 开始分析：xxx.pdf
     文件类型：PDF
     ...
```

## 错误处理

| 错误 | 处理 |
|------|------|
| 文件类型不支持 | 提示用户仅支持PDF/Word/Excel/Markdown |
| 文件过大（>50MB） | 提示文件过大，建议拆分 |
| 下载失败 | 重试3次，失败则提示用户 |
| 磁盘空间不足 | 提示清理空间 |

## 依赖

- 飞书SDK或API
- 文件系统操作权限
- 后续样本分析Skill
